### 2. Окно входа

#### Оптимизированный дизайн:
- Фиксированный размер: **600x800 пикселей**
- Ширина карточки: **380px**
- Заголовок "Вход в систему": 24px, хорошо читаемый
- Поля ввода: **50px высота**, 15px шрифт
- Кнопка "Войти": **42px высота**, на всю ширину
- Компактная тестовая информация с цветовыми акцентами

### 3. Расширение базы данных

#### Добавлены новые таблицы:

**attachments** - для хранения вложений к заявкам
```sql
- id (PK)
- request_id (FK → repair_requests)
- file_name
- file_path
- file_type
- file_size
- description
- uploaded_by (FK → users)
- uploaded_at
```

**notifications** - система уведомлений
```sql
- id (PK)
- user_id (FK → users)
- title
- message
- type (info/warning/success/error)
- is_read (0/1)
- related_request_id (FK → repair_requests)
- created_at
```

#### Добавлены индексы для оптимизации:
- `idx_attachments_request` - для быстрого поиска вложений по заявке
- `idx_notifications_user` - для быстрого получения уведомлений пользователя
- `idx_notifications_read` - для фильтрации непрочитанных
- `idx_audit_log_table` - для быстрого поиска в журнале аудита

### 4. Система уведомлений

#### Реализован UI:
- **Виджет в sidebar** с счетчиком непрочитанных уведомлений
- Иконка 🔔 + цифра + текст "новых"
- Динамическое изменение цвета счетчика (cyan при наличии новых)

#### Методы для работы:
- `_update_notifications_count()` - обновление счетчика
- `_create_notification(user_id, title, message, type, request_id)` - создание уведомления

### 5. Цветовая схема (темная тема)

#### Палитра как на референсах:
- **Фон:** #0F0F0F (очень темный)
- **Карточки:** #2A2A2A
- **Акценты:**
  - Cyan: #00FFFF (основной акцент)
  - Purple: #C77DFF (выделение, активные элементы)
  - Magenta: #FF00FF (критичные элементы)
  - Green: #00FF88 (успешные статусы)
  - Orange: #FF6B35 (предупреждения)

### 6. Тестовые данные

#### В БД автоматически создаются:
- 7 пользователей (все роли)
- 8 автомобилей разных марок
- 30 заявок на ремонт с разными статусами и приоритетами
- 20 видов запчастей
- Категории дефектов, типы ремонта, поставщики

## 📝 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### База данных:
- SQLite 3
- 13 таблиц (включая новые attachments и notifications)
- 14 индексов для оптимизации
- Внешние ключи включены (PRAGMA foreign_keys = ON)

### Технологии:
- Python 3.12+
- PyQt6 (GUI)
- matplotlib (графики)
- pandas (обработка данных)
- reportlab (экспорт PDF)

### Архитектура:
- Однофайловое приложение (3300+ строк)
- MVC-подобная структура
- Логирование всех действий
- Безопасное хеширование паролей (SHA256, рекомендуется bcrypt)
