# 🚀 Быстрый старт Carvix AutoManager

## 1. Установка зависимостей

### Вариант 1: Через requirements.txt
```bash
pip install -r requirements.txt
```

### Вариант 2: Через install_requirements.py
```bash
python install_requirements.py
```

### Вариант 3: Вручную
```bash
pip install PyQt6 matplotlib pandas reportlab
```

## 2. Запуск приложения

```bash
python carvix_app.py
```

## 4. Основные функции

### 📊 Дашборд
- Просмотр ключевых метрик
- Графики статусов автопарка
- Последние заявки

### 🔧 Заявки на ремонт
- Создание новых заявок
- Фильтрация по статусу/приоритету
- Назначение механиков
- Изменение статусов

### 🚗 Автопарк
- Список всех ТС
- Информация о пробеге
- Статусы автомобилей

### 📈 Аналитика
- KPI показатели
- Графики затрат
- Топ проблемных единиц
- Экспорт в Excel/PDF

### 👥 Пользователи (только админ)
- Добавление пользователей
- Назначение ролей
- Блокировка/разблокировка

### 📚 Справочники
- Виды ремонта
- Виды ТО
- Категории неисправностей
- Поставщики

### ⚙️ Запчасти
- Учет на складе
- Контроль остатков
- Привязка к поставщикам

## 5. Структура базы данных

При первом запуске автоматически создается файл:
- `carvix_database.db` - SQLite база данных

## 6. Логирование

Логи сохраняются в файл:
- `carvix_debug.log`

## 7. Экспорт данных

### Excel
- Требуется: `pandas`
- Доступен в разделе "Заявки" и "Аналитика"

### PDF
- Требуется: `reportlab`
- Доступен в разделе "Заявки" и "Аналитика"

## 8. Горячие клавиши

- `Enter` - подтверждение в диалогах
- `Esc` - отмена/закрытие диалога
- `Ctrl+Q` - выход из приложения

## 9. Устранение неполадок

### Ошибка "No module named 'PyQt6'"
```bash
pip install PyQt6
```

### Ошибка доступа к базе данных
Удалите файл `carvix_database.db` и перезапустите приложение

### Графики не отображаются
Установите matplotlib:
```bash
pip install matplotlib
```

## 10. Поддержка

При возникновении проблем:
1. Проверьте логи в файле `carvix_debug.log`
2. Убедитесь, что все зависимости установлены
3. Попробуйте удалить БД и перезапустить
